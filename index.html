The Basics

Welcome to The ElemenTest, where I hope to test out as many HTML elements as I can.

In this section, we begin with a paragraph element, a fundamental way to group text into a unit. The width the element is allowed determines the number of lines that appear if no line break is forced. It is known as a block-level element, as when it is used it gets put on its own "tier" so to say, and exits out of a line if it was initiated from one, into its own "block".

To initiate a second paragraph, the first <p> element is closed and a new one is created. This is the second one. To represent HTML tags within HTML, the Entity representation must be used. In this case an entity name was used (as opposed to an entity number). An interesting exercise that I should carry out later would be to alter the ordinal number word instances above (e.g., second) to match the nth <p> element, so should another paragraph get added before this one, the reference would still be correct.

To learn more about Entities, visit the W3 Schools page. The W3 Schools is a great reference, but not comprehensive enough to be used on its own. It should be supplemented by resources such as the Mozilla Developer Network (MDN) to add the lacking depth. The highlighting and links seen above are courtesy of inline-level elements. They exist, as their name indicates, inline wih the text they sit beside. Unlike the block-level elements, they don't create new lines just by existing, but are meant to alter the lines they reside in. Another example of a tags that can be used to achieve the same things that CSS does is the del tag. The second strikethrough in the previous sentence was done through CSS via the text-decoration: line-through style on a span selector. I should also add an add tag was used on the underlined a (which can also be achieved with the text-decoration: underline style), so the plural version was actually correct but the opportunity to use both tags was there and I took it :). The important distinction though is that using the appropriate semantic HTML tags improves accessibility, by letting software such as screenreaders know they should for example change the tone of voice when reaching that content. This does not happen with simply altering the visual appearance through CSS styles. Also worthy of note, there are tags that give the bold and italics appearance of the strong and em tags, but do not lend accessibility features. I've just used them; the i tag should be used when writing about foreign words, idiomatic phrases, technical terms, ship names, thoughts, etc. The b tag should be used to draw attention to a word without conveying importance. It is the least semantic of the tags. The use of these tags also helps with SEO as well as the previously mentioned boons to accessibility.

Semantic importance in order

  1. em - Changes the actual meaning of content, emphasizes text.
  2. strong - Doesn't change the meaning of content, indicates importance or seriousness.
  3. i - Doesn't change the meaning of content, indicates a special term.
  4. b - Doesn't change the meaning of content, draws attention to.

There are a number of different list attributes that can be used to affect their behavior. The start attribute changes what number an ordered list starts on, while the value attribute on a li element will change what number that item continues on. The reverse attribute (with no value, the ="" part) will reverse the numbering.

Another set of semantic tags are the sub and sup tags, which create subscripted and superscripted text respectively. For example, if 23 = 8, then log2 8 = 3. There is also a small tag but it differs in appearance from subscript text as it stays at the level of the main text.

Imagine your own private cabin in the woods, a quiet abode far away from noise of apartment life. A rarity in New York (at least the Southern tip) but a dream worth holding onto nonetheless.

The only creaking heard is the invited cracks of a settling house as the fireplace exhales; not poorly constructed foundation giving way to the pounding onslaught from a stone-footed trash panda.

The benefits of uninterrupted silence?

  • Peace. You can't put a price on that.
  • Clearer thoughts.
  • Increased productivity.
  • Getting in touch with oneself.
  • As stated above, no (well, to be fair, less) interruptions.

The downsides of uninterrupted silence?

  • The need to mask tinnitus.
  • Clearer thoughts -- Yes it's a double edged sword. Sometimes we use interruptions to avoid reflecting on difficult memories.
  • Losing track of time. Perhaps this should also be added to the benefits list though?
  • That uneasy feeling that every slight noise brings.
